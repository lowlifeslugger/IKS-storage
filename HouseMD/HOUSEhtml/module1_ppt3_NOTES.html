Introduction to Operating Systems


An operating system (OS) manages and coordinates all functions performed by computer hardware — including the CPU, I/O devices, secondary storage, and network equipment.


What the OS does:

- Recognizes input from keyboard; sends output to display screen.

- Keeps track of files and directories on disk.

- Controls peripheral devices (disk drives, printers).

- Tracks each hardware resource — determines who gets what, when, and terminates access.


Primary purpose: Maximize computer productivity by operating in the most efficient manner and minimizing human intervention.


> The OS must be loaded and activated before any other task. It is the most indispensable component of the software interface between users and hardware.


Examples: Windows, Linux, macOS.


---


Functions of an Operating System


# Function Description
1 Job Management Recognizes jobs, identifies priorities, checks memory availability, schedules, and runs each job at the appropriate time.
2 Batch Processing Data accumulated and processed in groups (not in real-time).
3 Online Processing Data processed instantaneously. *Example:* Salesperson checks if an item is in stock — online system instantly acknowledges the request.
4 Data Management Manages storage and retrieval of data.
5 Virtual Storage Increases effective main memory by breaking jobs into pages and keeping only a few in RAM at a time. Large jobs can be processed by a limited CPU.
6 I/O Management Manages data flow to/from the computer — among computers, terminals, printers, and other devices.

Classification of Operating Systems


Operating systems can be classified in two ways:

- By use: desktop, server, mainframe.

- By capabilities/features: multi-user, multiprocessing, multitasking, etc.


---


1. Desktop Operating Systems

- Run on personal desktop computers.

- Examples: Windows (currently leading), macOS, Linux.


---


2. Server Operating Systems

- Provides services, resources, and functionality to multiple users/client devices across a network.

- Optimized for database management, web services, and high security.

- Examples: Windows Server, UNIX-based systems, macOS Server.


---


3. Mainframe Operating Systems

- Designed for large-scale organizations with critical applications: bulk data processing, large-scale transactions, high-volume I/O.

- Optimized for high-volume, high-reliability, high-availability environments.

- Examples: z/OS (IBM), z/VM (IBM).


---


4. Multi-User Operating Systems

- Multiple users log in and work simultaneously — from different locations or terminals.

- OS efficiently manages CPU time, memory, disk space, and I/O devices fairly among all users.

- Examples: Windows Server, UNIX.


---


5. Multiprocessing Operating Systems

- Supports and manages multiple CPUs within one computer system.

- Executes multiple processes simultaneously — significant performance improvement.

- Tasks divided into sub-tasks and executed in parallel across multiple processors.


Key characteristics:

- Two or more CPUs sharing tasks.

- Parallel processing leads to faster execution.


---


6. Multitasking Operating Systems

- Ability to execute more than one program at the same time.

- CPU switches between programs very quickly.


Two types of multitasking:


Type Description
Preemptive multitasking OS can interrupt a running process, pause it, and allocate CPU to another process.
Cooperative multitasking Each program controls the CPU for as long as it needs; OS cannot forcibly interrupt.

---


7. Multithreading Operating Systems

- Allows different parts of a single program to run concurrently.

- These parts are called threads.


---


8. Real-Time Operating Systems (RTOS)

- Handle tasks with precise timing constraints.

- Provide guaranteed response times for critical operations.


Applications:


Domain Example
Aerospace & Defense Satellite systems, military applications
Automotive Vehicle control systems
Medical Devices Controlling life-critical equipment

Examples: CCP (Communication Control Programs), iRMX (Real-Time Multitasking Executive for Intel).
