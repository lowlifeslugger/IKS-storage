What is Python?


Python combines the elegant style of C with the class/object features of Java, but with far fewer lines of code than either.


# Python program to add two numbers
a = b = 10
print("Sum =", a + b)   # Output: Sum = 20

- Developed by Guido Van Rossum in 1991.

- Open source — freely downloadable from [www.python.org](http://www.python.org).

Features of Python


Feature Description
Simple Reads like English sentences. Easy to develop and understand.
Easy to learn Very few keywords; simple structure; resembles C. Migrating from C is easy.
Open source Free to download and use — no licensing cost.
High level language Uses English words; abstracts hardware details from the programmer.
Dynamically typed No type declarations. A name can be assigned to one type, then later to another.
Platform independent Works on UNIX, Linux, Windows, Mac — write once, run anywhere.
Portable Produces same result on any computer (platform independent).
Procedure & Object Oriented Supports both procedural (functions) and OOP (classes, objects, inheritance).
Interpreted Source code compiled to byte code, then executed line-by-line by PVM.
Extensible C/C++ programs can be integrated into Python and run using PVM.
Embeddable Python programs can be inserted into C/C++ programs.
Huge library Large standard library + third-party packages (numpy, w3lib, etc.).
Database connectivity Interfaces to connect Python programs to all major databases.
Scalable Programs scale across platforms and leverage new platform features.
Batteries included Pre-built packages immediately available. numpy for arrays, w3lib for web functions.

Execution of a Python Program


Source code  (.py)
      ↓
 Python Compiler
      ↓
  Byte code  (.pyc)
      ↓
PVM — Interpreter
      ↓
 Machine code  (0s and 1s)
      ↓
   Processor → Output

- Source code — Written by programmer as x.py.

- Byte code — Fixed set of instructions; 1 byte per instruction; stored in x.pyc.

- PVM (Python Virtual Machine) — Converts byte code to machine code using an interpreter; sends to processor.

- Interpreter — Translates source line by line; hence slower than compiled languages.

- JIT (Just-in-Time) Compiler — Sometimes added to PVM to improve execution speed.


> Note: All steps (source → byte code → machine code → output) are internally completed before the final result is displayed.

Memory Management in Python


- In C, programmers manually allocate/deallocate memory using malloc() and free().

- In Python, memory allocation and deallocation are done automatically by PVM.

- Everything in Python is an object — strings, functions, integers.

- Objects are stored in the heap. Heap size adjusts dynamically based on RAM availability.


Memory management layers:


Layer Role
Operating System Allocates RAM for any program.
Raw Memory Allocator Checks whether enough memory is available for storing objects.
Object-Specific Allocators Handle each type differently — e.g., integers stored differently from strings.

Garbage Collection in Python


The garbage collector (module gc) deletes objects from memory that are no longer used in the program.


How it works:

- Maintains a reference count for each object.

- When reference count = 0 → object is deleted from memory.

- Can detect and remove reference cycles.


Three Generations:


Generation Description
Generation 0 Newly created objects.
Generation 1 Objects that survived one garbage collection pass.
Generation 2 Long-lived, older objects.

- GC prefers deleting younger unreferenced objects over older ones.

- GC runs automatically based on a threshold value.

- In cases of reference cycles, it's better to run GC manually.
